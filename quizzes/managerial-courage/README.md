# שאלון אומץ - מנועי ההשפעה וההובלה

שאלון אבחון עצמי של מנועי ההשפעה וההובלה (Klein & Klein 2012).

---

## מבנה הקבצים

```
managerial-courage/
├── index.html      # HTML structure
├── styles.css      # Styles (imports ../shared.css)
├── app.js          # Logic only - no Hebrew strings
├── content.json    # All Hebrew text (questions, categories, descriptions)
└── README.md       # This file
```

**הפרדת תוכן:** כל הטקסט בעברית מאוחסן ב-`content.json`, כך ש-`app.js` מתמקד בלוגיקה בלבד.

---

## פורמט השאלות: סולם ליקרט

כל שאלה מציגה משפט אחד, והמשתמש בוחר ערך בסולם 1-5:

- **1** = בכלל לא
- **2** = במידה מועטה
- **3** = במידה בינונית
- **4** = במידה רבה
- **5** = במידה רבה מאוד

```json
{
  "id": 1,
  "text": "אני משרה אופטימיות ואנרגיות חיוביות על הסביבה שלי"
}
```

**UI:** משפט אחד במרכז, עם סולם 1-5 מתחתיו.

---

## ניקוד: קטגוריות לפי שאלות

השאלות מחולקות ל-5 קטגוריות (מנועי השפעה). כל קטגוריה כוללת 6 שאלות, והציון הוא סכום התשובות.

```json
{
  "id": "goal-orientation",
  "title": "מכוונות ליעדים",
  "questions": [3, 4, 8, 13, 23, 26],
  "description": "האומץ להציב ולהשיג יעדים שאפתניים",
  "scoreRange": [6, 30]
}
```

סכום של 6 שאלות → טווח ציון: 6-30

**קטגוריות:**
| ID | שם | שאלות | טווח ציון |
|----|-----|--------|-----------|
| goal-orientation | מכוונות ליעדים | 3,4,8,13,23,26 | 6-30 |
| inspiration | השראה והתלהבות | 1,6,11,16,21,29 | 6-30 |
| relationships | מחויבות ליחסים | 2,7,12,17,22,28 | 6-30 |
| openness | פתיחות | 10,15,19,20,25,27 | 6-30 |
| rigor | קפדנות ומשמעת | 5,9,14,18,24,30 | 6-30 |

## ניתוח: שלושה טווחים

הציונים מפורשים לפי שלושה טווחים:

| טווח ציון | תיאור |
|-----------|--------|
| 6-15 | נמוך - יש מקום לשיפור |
| 16-22 | בינוני - יש יכולת, ניתן לחזק |
| 23-30 | גבוה - מצטיין |

כל קטגוריה כוללת טקסט פרשנות מוכן לכל טווח, המאוחסן ב-`content.json` תחת `categories[].interpretation`.

## מסכים

1. **הנחיות** - הסבר על השאלון, סולם התשובות, כפתור התחלה
2. **שאלות** - שאלה אחת בכל פעם, סרגל התקדמות, כפתורי קדימה/אחורה
3. **תוצאות** - כרטיסי ציונים לפי קטגוריה (5 קטגוריות), הדגשת מנועים דומיננטיים
4. **ניתוח** - פירוט לפי קטגוריה עם פרשנות, זיהוי מנועים דומיננטיים ואזורי פיתוח

## הבדלים ממודל הפרואקטיביות

**דומה:**
- מבנה מסכים (הנחיות → שאלות → תוצאות → ניתוח)
- סרגל התקדמות
- מנגנון העתקה ללוח
- הפרדת תוכן (content.json)

**שונה:**
- פורמט שאלות: משפט יחיד עם סולם ליקרט (לא משפטים קוטביים)
- סולם: 1-5 עם משמעות ישירה (לא 1=ימין, 5=שמאל)
- ניקוד: סכום פשוט לפי קטגוריה (לא ניתוח תגובתי/פרואקטיבי)
- קטגוריות: 5 במקום 6
- זיהוי מנועים דומיננטיים (הציון הגבוה ביותר)
