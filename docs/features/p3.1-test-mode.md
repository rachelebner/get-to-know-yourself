# P3.1 Test Mode

Hub-level toggle for quick testing of all questionnaires.

## Behavior

When enabled:
1. Questionnaires auto-fill random answers on load
2. Skip directly to results/analysis screen
3. No manual input required

## UI

Hub header toggle:
- Toggle switch or button in hub header
- Visual indicator when active
- Persists via URL param `?testmode=1`

## Enhanced Copy (Test Mode)

When copying results in test mode, the output includes:

1. **Warning at top:**
   ```
   ⚠️ תוצאות בדיקה - דוגמה על בסיס מילוי אקראי
   ⚠️ לא תוצאות אמיתיות
   ```

2. **Q&A table at end:**
   ```
   ## פירוט התשובות (מילוי אקראי)
   | מספר שאלה | תשובה שנבחרה |
   |-----------|--------------|
   | 1 | 3 |
   | 2 | 5 |
   ...
   ```

This clearly marks test outputs as non-real and shows the randomly generated answers for verification.

## Implementation

`lib/testmode.js` exports:
```javascript
// Check if test mode is active
isTestMode() → boolean

// Get URL with test mode param
getTestModeUrl(baseUrl) → string

// Insert visual indicator in questionnaire header
insertTestModeIndicator()

// Update back-to-hub links to preserve test mode
updateBackLinks()
```

## Integration

Hub:
- Add toggle to header
- Pass `?testmode=1` to questionnaire links when active

Each questionnaire:
1. Import `lib/testmode.js`
2. Check `isTestMode()` on init
3. If true: fill random answers, show results immediately
4. In `buildResultsMarkdown()` / `buildResultsRichText()`: add warning + Q&A table when `isTestMode()` is true
